K-Means

K-Means

Learning Objectives

  • Distinguish between coordinate descent and block coordinate descent
  • Define an objective function that gives rise to a “good” clustering
  • Apply block coordinate descent to an objective function preferring each point to be close to its nearest objective function to obtain the KǦMeans algorithm
  • Implement the KǦMeans algorithm
  • Connect the nonconvexity of the KǦMeans objective function with the (possibly) poor performance of random initialization

Inductive Bias

Algorithm

Given unlabeled feature vectors $\mathcal{D} = {\mathbf{x}^i}_{i=1}^N$
Initialize cluster centers $c = {\mathbf{c}_1, \mathbf{c}_2,…,\mathbf{c}_K}$
and cluster assignments $\mathbf{z} = {z^1,z^2,…,z^N}$ for $\mathcal{D}$, i.e. $\mathbf{x}^i$ is in class i
Repeat reassignment until convergence

  • for $j$ in ${1,…,k}$, $\mathbf{c}_j=$ mean of all points assigned to cluster $j$
  • for $i$ in ${1,…,N}$, $z^i = $ index of cluster center nearest to $\mathbf{x}^i$

Optimization

Decision Rule: Assign each point $\mathbf{x}^i$ to its nearest cluster center $\mathbf{c}^j$

Objective function: This is Non-Convex
$$c = \underset{c}{\argmin}\sum_{i=1}^N\underset{z^i}{\min}|\mathbf{x}^{i}-\mathbf{c}{z^i}|^2_2 \tag{$\mathbf{x}^i$ has cluster assignment $z^i$}$$ can be rewritten as $$c, \mathbf{z} = \argmin{c,\mathbf{z}}\sum_{i=1}^N|\mathbf{x}^i-\mathbf{c}{z^i}|^2_2 = \argmin{c,\mathbf{z}}J(c,\mathbf{z})$$

Block Coordinate Descent

Repeat reassignment in practice (block coordinate descent):

  • $\mathbf{c} = \argmin_cJ(c,\mathbf{z})$, which decomposes in close form $$J(c,\mathbf{z}) = \sum_{i=1}^N|\mathbf{x}^i-\mathbf{c}{z^i}|^2_2 = \sum{j=1}^K\sum_{i:z^i=j}^N|\mathbf{x}^i-\mathbf{c}_j|^2_2$$ $$\mathbf{c}k = \argmin \sum{i:z^i=k}|\mathbf{x}^i-\mathbf{c}k|^2_2$$ $$= \frac{1}{N_k}\sum{i:z^i=k}\mathbf{x}^i \tag{intuitively just the average of all x}$$
  • $z_i = \argmin_{\mathbf{z}}J(c,\mathbf{z})= \underset{j}{\argmin}|\mathbf{x}^i-\mathbf{c}_j|^2_2$

What’s the difference with coordinate descent?
Block coordinate descent tries to minimize $J(\alpha, \beta)$ wiht two parameters, while coordinate descent tries to minimize $J(\theta)$ with one parameter.
In coordinate descent, we pick one dimension and minimize along that dimension. In block coordinate descent, we minimize over an entire group at a time. And update $\alpha$ while fixing $\beta$, update $\beta$ while fixing $\alpha$.

Initialization (matters a lot)

  • Random (Lyold’s Method): may converge to local optimum, can be arbitrarily bad
