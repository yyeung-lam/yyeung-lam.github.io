Sliding Window

Reusing common elements in a sliding window.

In some problems, the size of the sliding window is not fixed.

Smalllest subarray with given sum s (easy)

The size of sliding window is not fixed.

  1. Add up elements until the sum is greater or equal to s
  2. These element will constitute our first sliding window
  3. Keep adding elements, and try to shrink the size of sliding window until the sum is smaller than s. This shrinking will happen in multiple steps. In each step:
    1. Update smallest window length
    2. Subtract the first element of the window from the running sum to shrink the sliding window

Longest Substring with Same Letters after Replacement (hard)

Idea: Keep track of maximum frequency in the sliding window. Then the number of replacement = end - start + 1 - maximum frequency

Note: The max_freq need not be updated to a lesser value when shrinking the window size. Because we only care if the size can enlarge.
