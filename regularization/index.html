Feature Engineering

Non-Linear Features

Idea: let input be some function of $\mathbf{x} = [x_1, …, x_M]$
Examples: polynomial, sigmoid, log, …
Takeaway: Non-linear features allow linear models to capture nonlinear aspects of the original input without changes to the model

Regularization

Motivation: Occam’s Razor $\Rightarrow$ small number of (important) features

Given: Objective Function $J(\theta)$
Goal: Find $\hat{\theta} = \underset{\theta}\mathrm{argmin} \ J(\theta) + \lambda r(\theta)$
Def: Regularizer $r(\theta)$, tradeoff between fitting the data and keeping the model simple.

$q$ $r(\theta)$ yields name optimization notes
0 $|\theta|0 = \sum 1{(\theta_m\neq 0)}$ zeros L0 reg. bad, rarely used
1 $|\theta|_1 = \sum |\theta_m|$ zeros L1 reg. subdifferentiable, for sparse output, feature selection
2 $|\theta|2$$= \sum \theta_m^2$ small values L2 reg. differentiable, computational efficient, no sparse output

Remarks:

  • Don’t Regularize the Bias (Intercept) Parameter! (where $x_0 = 1$)

  • Whitening Data: by subtracting its mean and dividing by its variance, it helps all the features be penalized in the same units

  • If multinomial logistic regression is trained infinitely without regularization, the model pushes the probability to be more likely by pushing weights for correct labels towards infinite, and weights for incorrect labels towards negative infinite.
